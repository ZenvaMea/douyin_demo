# 📖 产品篇

## 一、一句话产品定位

> **粘贴抖音视频链接，AI 30 秒帮你拆穿营销号，逐句核查 + 引用权威源 + 真相版本。**

## 二、为什么需要打假搭子？

### 用户痛点

```
🔴 抖音 61% 健康类视频含夸大或错误信息
🔴 老妈群里每天转「专家说不能吃 XX」
🔴 「90% 的人不知道」「医生都震惊了」点开就掉坑
🔴 听信「稳赚不赔」结果套牢
🔴 自己百度查证太累，AI 生成又不可信
```

### 现有解法的不足

| 方案 | 问题 |
|------|------|
| 等专业账号辟谣 | 滞后、不一定刷到、不针对特定视频 |
| 自己搜索查证 | 累、不专业、搜出来还是营销号 |
| 看评论区 | 评论区也被刷，反而强化错误信息 |
| 普通 AI 总结 | 只会给摘要，不会给「真假判断 + 证据」 |

### 我们的差异化

> **不当裁判，只当陪审团 —— 不下结论，只摆证据。**

## 三、核心功能

### 🌟 1. 一眼看哪里有问题（Hero 功能）

**原文标注** — 在视频原文上**直接三色高亮**错误片段：

```
家人们，今天教你们一个绝绝子的养生方法！
每天早上空腹来一杯 ❌柠檬富含维C，是橙子的10倍❌
（红色下划线 + 划掉）
🤔加速代谢，帮你减肥🤔（黄色下划线）
```

点击红色文字 →「错误原文 vs 真相 vs 引用源」对比卡片

### 🎯 2. 三色核查报告

| 标签 | 含义 | 学术对应 |
|------|------|---------|
| 🟢 已验证 | 有权威证据支持 | SUPPORTED |
| 🟡 存疑 | 证据不足 / 有争议 | NEI（Not Enough Info） |
| 🔴 误导 | 与公认事实相悖 | REFUTED |

每条声明独立判定 + 5 星证据强度 + 引用源（卫健委 / 营养学会 / 医学期刊）+ 真相重写

### 📤 3. 转发家人群（信息丰富）

精美卡片 + 一键复制文字版：
- 顶部风险横幅（基于评分）
- 视频信息 + 主题概括
- 评分 + 进度条
- 三色统计
- 多条警示（最多 3 条）+ 引用源
- 一键复制 / 长按截图

### 📄 4. 导出 PDF 报告

杂志风简约设计 → 浏览器原生 print → 完美中文 PDF：
- 单条导出
- 批量导出（按筛选条件）
- 文字可复制可搜索（不是图片）

### 📚 5. 我的核查仓库

- localStorage 持久化（最多 50 条）
- 三色筛选：全部 / 含误导 / 含存疑 / 全靠谱
- 一键回看（**不重新调 API**，省钱省时）
- 单条 / 批量 PDF 导出
- 隐私优先：仅本地，不上传

### 🎮 6. 游戏化钩子（产品粘性）

- **🛡️ 防骗等级**：6 级体系（🐣→🦉→🔍→🛡️→⚔️→👑）
- **🎯 互动 Quiz**：「这 5 条常识你信过几条？」
- **🏛️ 谣言博物馆**：12 条经典谣言卡片墙（5 大分类）
- **📊 实时数据**：已核查 / 揪出谣言 / 帮人避坑（数字爬升动画）

## 六、核心数据流

```
用户粘贴抖音链接
    ↓
[1] 短链 302 跟随 → 提取 19 位 aweme_id
    ↓
[2] 抓 share 页 HTML（绕过 X-Bogus 签名）→ desc / 视频URL
    ↓
[3] desc 太短？→ 下载视频 → ffmpeg 抽 mp3 → 豆包 ASR 转写
    ↓
[4] Claimify 四阶段拆解 → 原子声明列表（带 original_quote）
    ↓
[5] 并发核查每条声明 → 三色 + 证据 + 真相重写（SSE 流式）
    ↓
[6] 前端渲染：
    ├── 原文标注（点击红字看真相）
    ├── 评分环 + 三色统计
    ├── 详细核查列表（可展开）
    └── 自动保存到本地仓库
```

## 七、关键技术决策

### 1. LLM 抽象层（多模型可切换）

**5 家 Provider 适配器，业务代码 0 改动切换：**

```typescript
// 切换豆包 ↔ Claude 只需改环境变量
LLM_PROVIDER=doubao  // anthropic / deepseek / qwen / kimi

// 业务代码无感
const provider = createProvider();
const result = await provider.chat(request);
```

设计原则：以 OpenAI Chat Completions 为「中间表示」，每家 < 50 行薄适配器。

### 2. 抖音内容获取（绕过 X-Bogus 签名）

**实战发现：** 2025 年 iesdouyin 的 `iteminfo` API 已要求 X-Bogus 签名。

**解决方案：** 直接抓 `share/video/{id}/` 页面的 HTML，从 SSR 注入的 `<script>` 数据里正则提取 `desc`、`nickname`、`play_addr`。

```typescript
// 关键正则
const descMatch = html.match(/"desc"\s*:\s*"((?:[^"\\]|\\.)*)"/);
const playAddrMatch = html.match(
  /"play_addr"\s*:\s*\{[^}]*?"uri"\s*:\s*"([^"]+)"[^}]*?"url_list"\s*:\s*\["([^"]+)"/
);
// playwm → play 转无水印
videoUrl = playwmUrl.replace('/playwm/', '/play/');
```

### 3. ASR 语音转写（复用同一 ARK Key）

**调研发现：** 火山方舟 ARK 没有独立 ASR 端点，但**多模态豆包**支持 `input_audio`！

**实施：**
```
视频 mp4 (11MB)
  ↓ ffmpeg-static (内置二进制，无需用户安装)
mp3 (16kHz / 单声道 / 64kbps)
  ↓ base64 编码
豆包 chat/completions + input_audio
  ↓
完整文字转写（实测 683 字精准）
```

**优势：** 复用同一个 `ark-key`，零额外开通。

### 4. SSE 流式核查（实时反馈）

```typescript
// 后端按事件流推送
event: meta          → { provider, model }
event: phase         → { phase: 'extracting', label }
event: extraction    → { claims: [...] }
event: phase         → { phase: 'verifying', total }
event: verification  → { progress, total, result }  // 每条单独推送
event: done          → { ok }

// 前端流式渲染：声明逐条出现 spinner，核查完成立刻变彩色
```

### 5. 原文标注算法（核心创新）

**用户视角升级：** 从抽象 C1/C2 列表 → 原文上直接标红

```typescript
// 1. 利用 AI 提取的 original_quote
matches = claims.map(c => ({
  start: transcript.indexOf(c.original_quote),
  end: ...,
  verdict: verifications.get(c.id).verdict,
}));

// 2. 排序 + 去重叠
filtered = matches.sort((a,b) => a.start - b.start)
                  .filter(no_overlap);

// 3. 切分原文为 segments
segments = build_text_mark_segments(transcript, filtered);

// 4. 渲染：text 普通显示，mark 三色高亮
```

### 6. PDF 导出（React Portal + 浏览器原生 print）

**踩坑：** 报告 div 嵌套在父容器内，被 `@media print` 的 visibility 规则误隐藏

**解决：** React Portal 渲染到 `document.body` 直接子节点

```typescript
return createPortal(
  <div className="report-print-root">...</div>,
  document.body,  // 关键：脱离 React 树挂到 body
);
```

**优势：** 零依赖、完美中文、文字可复制可搜索

## 八、Prompt 工程亮点

### Claimify 四阶段拆解（基于 Microsoft 2025 SOTA）

```
1. Selection（选择）        → 只保留可验证的事实主张
2. Disambiguation（消歧）   → 替换代词 / 补全省略
3. Decomposition（原子化）  → 复合句拆为独立声明
4. 中文增强                 → 营销话术过滤、设问句转换
```

中文场景特殊处理：
- 「家人们」「绝绝子」→ 删除（marketing）
- 「99% 的人不知道」→ 保留（这本身可核查）
- 「你知道为什么 X 吗？因为 Y」→ 「X 的原因是 Y」

### 核查 Prompt（强制 quote + URL）

```
1. 三色标签 + 5 星证据强度
2. 必须给出具体机构名（中国营养学会 / 卫健委 / WHO...）
3. 必须区分语境（柠檬有维 C ≠ 柠檬维 C 含量高）
4. 不确定时强制选 NEI（防止脑补）
5. 真相重写 3-5 句话点睛
```
