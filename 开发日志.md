## 1. 项目背景

### 1.1 赛题选择

抖音精选赛道方向一，三个可能切入点：「理解 / 表达 / 使用」。

我们选**「理解」** —— 因为短视频最大的痛点是「**真假难辨**」，而不是「看不懂」。

### 1.2 产品定位演进

- ❌ **第一版构思**：LifeKit（把视频变成可调用的工具包）—— 太宽泛，演示困难
- ✅ **最终版**：打假搭子 —— 痛点尖锐（营销号毒瘤），价值清晰（避免被骗），Demo 直观（红绿灯）

### 1.3 关键设计原则

1. **不当裁判，只当陪审团** —— AI 没有权威性，只展示证据让用户自己判断
2. **声明拆解，逐条核查** —— 不评价整个视频，只评价可独立验证的"原子声明"
3. **三色标签 = SOTA 命名** —— 🟢 SUPPORTED / 🟡 NEI / 🔴 REFUTED（学术对应）
4. **强制信源 quote** —— 防止 AI 脑补，每条结论必须明确机构名

## 2. 关键架构决策（为什么这么选）

| 决策 | 选择 | 原因 | 备选方案（弃用） |
|------|------|------|----------------|
| **后端 LLM** | 火山方舟豆包（doubao-seed-2.0-mini） | 抖音赛事必选 + OpenAI 兼容 + 复用 ark-key 调 ASR | Claude / DeepSeek（无 ASR） |
| **LLM 抽象层** | OpenAI Chat Completions 为「中间表示」 | 国产 6 家全部兼容，每家适配器 < 50 行 | LangChain（太重） |
| **结构化输出** | 统一走 tool 调用模拟 JSON Schema | 跨厂商最稳。豆包 / Kimi 不支持 `response_format.json_schema` | 各家原生 json_schema（不通用） |
| **抖音文案获取** | 直接抓 `share/video/{id}/` 页面 HTML 正则提取 | 旧 iteminfo 接口在 2025 末已要求 X-Bogus 签名 | iteminfo / X-Bogus 逆向（坑深） |
| **视频转文字** | 豆包多模态 `input_audio`（同一 ark-key） | 零额外开通、零成本试点 | 火山 Seed-ASR（要单独开通+鉴权）/ Whisper（要付费） |
| **ffmpeg** | `ffmpeg-static` npm 包内置二进制 | 用户无需 `brew install ffmpeg` | 系统 ffmpeg（依赖运维） |
| **前端框架** | Next.js 16 + App Router | 同进程跑后端 API + SSR + Turbopack 快 | Vite + Express（要俩进程） |
| **流式核查** | Server-Sent Events (SSE) | 简单可靠，浏览器原生支持 | WebSocket（过度设计） |
| **UI 设计语言** | 多邻国 Duolingo 风 | 活泼、3D 按钮、彩色卡片对应三色标签天然契合 | 苹果风（用户 50% 路上改了主意😅） |

## 4. 踩过的坑（避免接手者重复踩）

### 坑 ①：豆包不支持 `response_format.json_schema`

**现象：** 调用结构化输出报 `400: json_schema is not supported`
**原因：** 豆包只支持 `json_object`，不像 OpenAI 支持 `json_schema`
**修复：** 在 `src/lib/llm/openai-compatible.ts` 里**统一走 tool 调用模拟** JSON Schema，跨厂商最稳
**位置：** `openai-compatible.ts` 的 `buildRequest()` 方法

### 坑 ②：iesdouyin iteminfo 接口要求 X-Bogus 签名

**现象：** 调 `iteminfo?item_ids=xxx` 返回 `status_code: 11110, "encrypt_data_miss"`
**原因：** 抖音 2025 末升级了反爬，所有 web API 强制要求 X-Bogus / a_bogus 签名
**修复：** **绕开 API，直接抓 `share/video/{id}/` 页面 HTML**，从 SSR 注入的 `<script>` 数据里正则提取 desc / nickname / play_addr
**位置：** `src/lib/extractors/douyin.ts` 的 `fetchSharePage()` 方法
**风险：** HTML 结构会随版本变化，已用多个备选正则。生产环境建议加监控

### 坑 ③：ffmpeg-static 在 Next.js Turbopack 下路径错乱

**现象：** ASR 报 `ENOENT: spawn /ROOT/node_modules/ffmpeg-static/ffmpeg`
**原因：** Turbopack 把 `__dirname` 改写为虚拟路径 `/ROOT/`，但 ffmpeg 二进制在真实 node_modules
**修复：** 在 `next.config.ts` 加 `serverExternalPackages: ['ffmpeg-static', 'fluent-ffmpeg']`，让 Next.js 不打包它们
**位置：** `next.config.ts`

### 坑 ④：纯字幕 + BGM 视频 ASR 转出"无中文语音内容"

**现象：** 某些抖音视频用字幕滚动 + 背景音乐，没有人声
**修复：** 暂时无解（要 OCR 视频帧字幕），已在 README 标注为「~15% 视频不适用」
**TODO：** 后续接入 OCR（火山方舟同样的多模态接口可以传 image_url）

### 坑 ⑤：globals.css `@import` 顺序

**现象：** Tailwind 4 报 `@import rules must precede all rules aside from @charset and @layer`
**原因：** 把 Google Fonts 的 `@import url(...)` 放在了 `@theme {}` 之后
**修复：** **所有 `@import` 必须放在文件最顶部**
**位置：** `src/app/globals.css` 第 1-2 行

### 坑 ⑥：Next.js 不接受 `.js` 扩展名 import

**现象：** `Module not found: Can't resolve '@/lib/llm/index.js'`
**原因：** 实际文件是 `.ts`，Next.js bundler 要求扩展名匹配真实文件
**修复：** **全项目 `.js` import 替换为 `.ts`**（一次性 sed 处理）
**位置：** 所有 `src/lib/**/*.ts` 和 API 路由

### 坑 ⑦：豆包 ASR 触发阈值

**现象：** desc = 52 字时没触发 ASR，但内容主要在视频里
**原因：** 阈值 30 太低，应该按"去掉话题标签后的纯文本"判断
**修复：** 改用 `pureDesc = caption.replace(/#\S+/g, '').replace(/\s+/g, '')`，阈值提高到 80
**位置：** `src/app/api/extract/route.ts`

### 坑 ⑧：npm 缓存权限问题（macOS）

**现象：** `npm install` 报 `EACCES: previous npm versions cache permissions`
**修复：** 用临时 cache：`npm install --cache=/tmp/npmc-$$`
**永久方案（用户跑一次即可）：** `sudo chown -R 501:20 ~/.npm`

## 6. 核心代码导览

### 6.1 LLM 抽象层（`src/lib/llm/`）

接手时关心切换 LLM？看这里。

**类型定义：** `types.ts` —— 所有适配器实现 `LLMProvider` 接口

**OpenAI 兼容基类：** `openai-compatible.ts` —— 国产 5 家共用，差异通过子类的 `super({ baseURL, model, capabilities })` 表达

**关键方法：**
- `buildRequest()` —— 把统一 ChatRequest 翻译成 OpenAI 格式
- `parseResponse()` —— 把 OpenAI 响应翻译回统一 ChatResponse
- **JSON Schema 输出**：所有 OpenAI 兼容厂商**统一走 tool 模拟**（跨厂商最稳）

**工厂方法：** `index.ts` 的 `createProvider()` 按 `LLM_PROVIDER` env 切换

### 6.2 抖音解析（`src/lib/extractors/douyin.ts`）

接手时遇到提取失败？看这里。

**核心流程：**
1. `extractUrlFromText()` —— 从用户粘贴的文本里提取 URL
2. `followRedirect()` —— 跟随 v.douyin.com 短链跳转
3. `extractAwemeId()` —— 正则 `/(\d{15,})/` 提取 19 位 ID
4. `fetchSharePage()` —— **绕开 API，抓 HTML 正则提取**
   - desc / nickname / cover / play_addr 都在 HTML SSR 数据里
   - URL 里 `playwm` 替换为 `play` 拿无水印版

**坑：** HTML 结构会变，已用多个备选正则。如果失败，看 console.log 的 `[douyin debug]` 日志找规律。

### 6.3 ASR 服务（`src/lib/services/asr.ts`）

接手时调整 ASR 行为？看这里。

**三步：**
1. `downloadVideo()` —— fetch 视频写到临时文件
2. `extractAudio()` —— spawn ffmpeg 抽 mp3（16kHz 单声道 64kbps，最省 token）
3. `transcribeViaArk()` —— base64 编码 → 豆包 `chat/completions` + `input_audio`

**计费提示：** 豆包多模态按 token 计费，60 秒视频约 ~50 KB mp3 → ~70K token，单次 ~0.5 元

### 6.4 SSE 流式核查（`src/app/api/check/route.ts`）

接手时改变前端展示节奏？看这里。

**事件序列：**
```
event: meta          → { provider, model }
event: phase         → { phase: 'extracting'|'verifying', label, total? }
event: extraction    → { summary, claims[], discarded_segments[] }
event: verification  → { progress, total, result } (每条声明独立推送)
event: done          → { ok }
event: error         → { message }
```

**前端订阅：** `src/app/page.tsx` 里的 `runCheck()` 用 `ReadableStream` reader 解析。

### 6.5 主页（`src/app/page.tsx`）

接手时改 UI / 交互？看这里。

**状态机：**
```
idle → fetching → extracting → verifying → done
              ↓ (任意阶段失败)
            error
```

**关键 state：**
- `mode: 'link' | 'paste'` —— Tab 切换
- `extraction` —— 拆解结果
- `verifications: Map<claim_id, ClaimCardData>` —— 核查结果（按 ID 索引，便于流式更新）
- `progress: { done, total }` —— 核查进度

## 7. 环境配置

### 7.1 系统要求

- **Node.js**: ≥ 22（实测 v25.8.1，因为用了 `allowImportingTsExtensions`）
- **macOS / Linux** —— Windows 未测试（ffmpeg-static 应该也支持）
- **网络**：能访问 GitHub（npm install 需要）+ 能访问 ark.cn-beijing.volces.com

### 7.2 必填环境变量（`.env`）

```bash
LLM_PROVIDER=doubao
DOUBAO_API_KEY=ark-xxx        # ★ 必填，火山方舟控制台获取
DOUBAO_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
DOUBAO_MODEL=doubao-seed-2-0-mini-260428   # 多模态推理接入点 ID
```

### 7.3 ARK key 怎么获取

1. 登录 https://console.volcengine.com/ark
2. 「在线推理」→「自定义推理接入点」
3. 选模型：**doubao-seed-2.0-mini** 或 **doubao-seed-1.6-flash**（必须是支持音频的多模态模型）
4. 创建后拿到 endpoint ID（格式 `doubao-seed-x-x-xxx-yyy`），填到 `DOUBAO_MODEL`
5. 「API Key 管理」创建 key（格式 `ark-xxx`），填到 `DOUBAO_API_KEY`

### 7.4 启动

```bash
npm install
cp .env.example .env  # 编辑填 key
npm run dev           # → http://localhost:3000
```

### 7.5 一键 demo（CLI 版）

```bash
npm run demo:health     # 柠檬水谣言（6 条声明）
npm run demo:finance    # 基金推销（8 条声明）
npm run demo:food       # 食物相克（5 条声明）
```

## 10. 接手者 Checklist

按顺序做：

- [ ] **Step 1：能跑起来**
  - [ ] `git clone git@github.com:ZenvaMea/douyin_demo.git`
  - [ ] `npm install`
  - [ ] `cp .env.example .env` 并填入你自己的 ark-key
  - [ ] `npm run dev` 看到 `http://localhost:3000` 能打开

- [ ] **Step 2：跑通核心链路**
  - [ ] CLI demo：`npm run demo:health` 能看到三色卡片
  - [ ] Web 粘贴文案：粘贴 `src/samples/health.ts` 的内容能跑通
  - [ ] Web 抖音链接：粘贴 `https://v.douyin.com/sjLnwb9CoL4/` 能跑通 ASR 全流程

- [ ] **Step 3：理解架构**
  - [ ] 读 `src/lib/llm/types.ts` —— 理解抽象层
  - [ ] 读 `src/lib/extractors/douyin.ts` —— 理解抖音解析
  - [ ] 读 `src/lib/services/asr.ts` —— 理解 ASR 链路
  - [ ] 读 `src/app/api/check/route.ts` —— 理解 SSE 协议

- [ ] **Step 4：选第一个 P0 任务动手**
  - 推荐：`/api/extract` 改 SSE 流式（用户体验提升最大）
