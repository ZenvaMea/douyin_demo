### 2.3 接口设计的关键决策

**决策 1：以 OpenAI Chat Completions 作为「中间表示」**
- 原因：6 家国产模型全部 OpenAI 兼容
- 收益：每家适配器 < 50 行
- 例外：Anthropic 用官方 SDK 走自己的 Messages API

**决策 2：用 tool_use 模拟 JSON Schema 输出**
- 原因：跨厂商 JSON 输出实现差异大
- Anthropic：用 `respond_with_json` tool 强约束
- OpenAI 兼容：优先用 `response_format: json_schema`，否则降级到 tool 模拟

**决策 3：把 Web Search 抽成显式 capability**
- 原因：各家实现完全不同（Claude 原生 / Kimi `$web_search` / DeepSeek 自实现）
- 后续接入：根据 `provider.capabilities.webSearchNative` 自动选择路径

## 三、Prompt 工程验证

### 3.1 声明拆解（Claimify 中文优化版）

基于 Microsoft Claimify 2025 论文的四阶段 Prompt：
1. **Selection** — 只保留可验证事实主张
2. **Disambiguation** — 消除代词、补全主语
3. **Decomposition** — 复合句拆分为原子声明
4. **中文增强** — 营销话术过滤、设问句转换

输出 JSON Schema（强约束）包括：
- `claims[]`：原子声明 + 原文 quote + 领域 + 优先级 + 搜索关键词
- `discarded_segments[]`：透明展示丢弃的内容

### 3.2 单条核查（三色标签 + 证据）

强制输出结构化结果，包括：
- `verdict`: SUPPORTED / NEI / REFUTED
- `confidence`: 1-5 星证据强度
- `evidence[]`：每条证据必须包含 source_type + source_name
- `truth_rewrite`：3-5 句科普重写
- `caveats[]`：前提/例外/适用人群
- `needs_web_search`：是否建议进一步联网

### 7.3 模型表现亮点（真实输出节选）

#### ✅ 数据级精准核查
> 声明 C1：「柠檬维C是橙子的10倍」
> 🔴 **误导** ⭐⭐⭐⭐
> 推理：每100g 柠檬维C 约 22mg，每100g 橙子约 33mg，柠檬仅为橙子的约 67%。
> 引用：中国食物成分表

#### ✅ 学术深度（剂量阈值意识）
> 声明 C2：「海鲜+维C=砒霜」
> 🔴 **误导** ⭐⭐⭐⭐⭐
> 推理：海鲜中五价砷在大量维C作用下可转化为三价砷，**但需达到中毒剂量**，日常饮食远达不到该剂量。

#### ✅ 区分语境的中肯判断
> 声明 C4：「鸡蛋+豆浆=营养破坏」
> 🟡 **存疑** ⭐⭐⭐
> 推理：**生**豆浆含抗营养因子，但煮熟后即被破坏，正常饮食搭配安全。

#### ✅ 严谨的存疑处理
> 财经场景所有「某基金 XX」均判定为黄色，要求联网检索具体信息，避免无依据的负面判断。
