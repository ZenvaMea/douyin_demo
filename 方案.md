# 🛡️ 打假搭子 · 视频内容核查器

> 别让营销号偷走你的判断力——把每一条短视频，都拆成可验证的证据

**赛道：** 抖音精选 · 内容重构
**方向：** 从「理解」出发 · 让分散甚至矛盾的内容，变得更清晰、更具判断价值

---

## 一、核心洞察

### 用户真实困境

抖音精选里充斥着大量「看起来很有道理」的内容：

- 💊 「每天一杯柠檬水排毒减肥」
- 🍎 「西瓜和桃子一起吃会中毒」
- 💰 「××基金稳赚不赔，跟车上车」
- 🧬 「转基因食品致癌」

用户的困境不是**看不懂**，而是：

```
看完信了 → 转发了 → 实践了 → 被坑了 → 才发现是假的
```

### 现有解法的缺陷

| 解法 | 问题 |
|------|------|
| 自己搜索查证 | 累、不会、搜出来还是营销号 |
| 看评论区 | 评论区也被刷，反而强化错误信息 |
| 等专业账号辟谣 | 滞后、不一定刷到、不一定针对你看的那条 |

### 我们的机会

> **AI 是天然的「证据汇聚器」**——它能在 30 秒内做完用户需要 2 小时才能完成的多源核查。

---

## 二、产品定位

### 一句话定义

**输入一条抖音视频链接，输出一份可信的「内容核查报告」 + 真正的科普版本。**

### 不做什么（重要）

- ❌ 不做「真假二分裁判」——AI 没有这个权威
- ❌ 不直接给账号打「营销号」标签——法律风险
- ❌ 不替用户下结论——还判断权给用户

### 要做什么

- ✅ 把视频中的**主张**拆成一条条**可验证的声明**
- ✅ 用**权威信源**逐条交叉核验
- ✅ 用**证据等级**透明展示，让用户自己判断
- ✅ 提供一份**基于可信源重写**的科普版本

## 四、核心功能设计

### 4.1 一键核查流程

```
┌─────────────────────────────────┐
│  用户：粘贴抖音视频链接            │
└────────────┬────────────────────┘
             ↓
┌─────────────────────────────────┐
│  系统：提取视频文案/字幕           │
└────────────┬────────────────────┘
             ↓
┌─────────────────────────────────┐
│  AI：拆解为可验证的「声明列表」    │
│  例：                            │
│  ① 柠檬水能排毒                  │
│  ② 柠檬水能减肥                  │
│  ③ 早晨空腹喝最有效              │
└────────────┬────────────────────┘
             ↓
┌─────────────────────────────────┐
│  AI + 联网搜索：逐条核查          │
│  ├── 学术论文                    │
│  ├── 官方机构                    │
│  ├── 主流媒体                    │
│  └── 专业自媒体                  │
└────────────┬────────────────────┘
             ↓
┌─────────────────────────────────┐
│  输出：分级核查报告 + 科普重写版    │
└─────────────────────────────────┘
```

### 4.2 「声明卡片」三色标签

每条声明独立展示，互不掩盖：

#### 🟢 已验证 · 有权威证据支持

```
声明：「维生素C有助于胶原蛋白合成」

✅ 证据强度：⭐⭐⭐⭐⭐
📚 引用：
  - 中国营养学会《维生素 C 营养指南》
  - WHO 官方数据
🔗 延伸阅读：[原文链接]
```

#### 🟡 存疑 · 证据不足或有争议

```
声明：「每天 8 杯水最健康」

⚠️ 证据强度：⭐⭐
📚 现状：
  - 实际饮水量取决于体重、活动、气候
  - 「8 杯水」无统一医学共识
  - 可能存在过度饮水风险
🔗 不同观点：[哈佛医学院] [WHO] [中国膳食指南]
```

#### 🔴 误导 · 与公认事实相悖

```
声明：「西瓜和桃子一起吃会中毒」

❌ 证据强度：⭐
📚 核查结果：
  - 中国食品科学技术学会：无毒性反应记录
  - 谣言溯源：源于 2014 年微信谣言
🔗 辟谣源：[中国食品辟谣网] [科普中国]
```

### 4.3 「真相版本」：基于可信源的内容重写

不只是给出错误判断，更要**给出正确版本**：

```
原视频（节选）：
「每天空腹喝柠檬水，排毒减肥又美白！」

🔄 真相版本：
「柠檬水 ≈ 水 + 微量维C
- ❌ 排毒：人体排毒靠肝肾，柠檬不参与
- ❌ 减肥：减肥靠热量缺口，与柠檬无关
- ⚠️ 美白：维C 有助合成胶原，但柠檬维C含量低
- ⚠️ 空腹：胃酸敏感者可能引发不适

✅ 真实建议：
喝水补水即可，柠檬作为风味添加无害但也无神效。」

📎 引用源：[中国营养学会] [Mayo Clinic]
```

### 4.4 「同主题对照」：跨视频证据汇总

同一主题刷到多条不同观点的视频时，AI 自动整合：

```
主题：「鸡蛋胆固醇」

📊 三方观点对照：

视频A（健康博主）：「每天最多吃1个蛋」
  └─ 证据：⭐⭐ 基于 1968 年旧研究

视频B（营养师）：「健康人每天 2-3 个无问题」
  └─ 证据：⭐⭐⭐⭐ 基于近 10 年新研究

视频C（养生号）：「鸡蛋是胆固醇炸弹」
  └─ 证据：⭐ 制造焦虑，无依据

🎯 当前主流共识：
健康人群每天 1-2 个全蛋，对血胆固醇影响有限。
有心血管疾病者建议咨询医生。
```

### 技术选型

| 层级 | 技术方案 | 选型理由 |
|------|---------|---------|
| 前端 | Next.js 15 + Tailwind CSS + shadcn/ui | 快速搭建、组件成熟 |
| AI 核心 | Claude API（claude-sonnet-4-6） | 长上下文 + 强结构化输出 + 工具调用 |
| 联网核查 | Claude Web Search Tool | 官方原生支持，引用可追溯 |
| 字幕提取 | 优先用户粘贴 + 后期接 ASR | Demo 阶段降低复杂度 |
| 提示词 | 启用 Prompt Caching | 核查模板复用，降低成本 |
| 存储 | LocalStorage / IndexedDB | 历史核查记录 |

## 六、Demo 体验设计

### Demo 路径（3 分钟完整演示）

#### 场景一：健康类谣言核查

1. 粘贴一条「每天空腹喝柠檬水排毒减肥」的视频链接
2. 系统拆解出 4 条声明
3. 实时联网核查，逐条标注 🔴🟡🟢
4. 输出科普重写版本 + 完整引用链

#### 场景二：争议话题对照

1. 粘贴 3 条关于「鸡蛋每天吃几个」的不同观点视频
2. 系统识别核心争议点
3. 生成对照报告 + 当前主流共识

#### 场景三：财经避坑

1. 粘贴一条「××基金稳赚」的推荐视频
2. 系统识别出投资风险声明
3. 标注证据强度，输出风险提示

## 七、差异化亮点

| 对比维度 | 现有方案 | 打假搭子 |
|---------|---------|---------|
| **核心动作** | 摘要、笔记 | 核查、辨真 |

| **社会价值** | 提效 | 反信息污染 |

## 八、商业 / 社会价值

### 短期价值

- **个人**：避免被谣言伤害（健康、财产）
- **平台**：提升内容生态质量

### 长期想象

- **数据沉淀**：高质量的「核查图谱」可成为反谣言基础设施
- **协作扩展**：用户可贡献核查结果，形成众包知识库
- **能力开放**：API 可输出给其他内容平台使用

## 九、风险与应对

| 风险 | 应对策略 |
|------|---------|
| AI 核查结果出错 | 强制展示信源链接，让用户可验证 |
| 被指控诽谤特定账号 | 不针对账号，只针对「声明」 |
| 权威源本身有误 | 多源交叉，标注共识程度 |
| 时效性问题（旧研究被推翻） | 标注研究年份，优先近 5 年文献 |
| 灰色话题（中医、玄学） | 明确标注「领域共识不统一」 |

## 十二、为什么是这个方案

> **抖音精选最大的痛点不是「内容看不懂」，而是「内容真假难辨」。**
>
> 我们不让 AI 替用户思考，而是让 AI 替用户做最累的那一步——**收集证据**。
>
> 把判断权还给用户，把信源透明给用户，让短视频从「被动信息灌输」，变成「主动证据决策」。

**核心一句话：**
> **刷之前，先核一下。**
